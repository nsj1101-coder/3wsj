EMR 서비스 개요

EMR(Electronic Medical Record)은 의료법 제22조 및 제23조에 근거한 전자의무기록시스템으로, 환자의 진료정보를 전자적으로 생성, 관리, 보관하는 시스템입니다.

15
개발 완료 DB 모델
⚙

법적 근거

「의료법」 제22조(진료기록부 등), 제23조(전자의무기록), 같은 법 시행규칙 제14조에 의거하여 전자의무기록시스템 제품인증 제도를 운영

⚖

인증 유형

유형1 의원급 / 유형2 병원급(300병상 미만) / 유형3 종합병원(300병상 이상). 모든 유형에서 필수 기능 항목은 동일하게 적용

✓

인증 절차

인증 신청 → 명세서/자가점검 결과서 제출 → 현장 심사(1주) → 인증 결과 통보. 재단법인 한국보건의료정보원장에게 신청

EMR 인증의 6대 분류 체계
  • 1. 진료정보관리 — 환자등록, 처방, 임상정보 관리
  • 2. 의료기관 정보공유 — 진료의뢰, 회송, 타기관 연동
  • 3. 환자 정보제공 — 환자 포털, 정보 열람 제공
  • 4. 보안관리 — 접근권한, 암호화, 접속기록, 전자서명
  • 5. 표준관리 — KCD, EDI, LOINC, SNOMED CT 연계
  • 6. 상호운용성 — HL7 FHIR, 건강정보 교류 표준

STRENGTH

베가스 EMR — 풍부한 메뉴 구성

업계 1위. 다양한 기능이 체계적으로 메뉴화, 오랜 기간 고도화된 것이 체감됨. CRM 기능이 특히 우수하여 환자 관리/마케팅/리텐션 영역이 눈에 띄는 수준

WEAKNESS

전 업체 공통 — 노후한 UI/UX

구 Windows 스타일 인터페이스. 기능은 많으나 시각적 현대화 부족. 자사의 모던 웹 기반 UI는 강력한 차별화 포인트

REFERENCE

상단 메뉴 네비게이션

기존 참고 레퍼런스는 좌측 메뉴였으나, 베가스는 상단 메뉴 구조 채택. 메뉴 구조 선택은 전략적 판단 필요

REFERENCE

하드웨어 연동 생태계

키오스크, 영수증 프린터, 카드 리더기 등 외부 디바이스 연동 지원. 대기 TV 연동은 모든 업체 공통 필수 기능으로 확인

OPPORTUNITY

모던 UI/UX 차별화 기회

모든 경쟁사가 구형 UI. 현대적 웹 기반 디자인으로 접근하면 강력한 시장 차별점 확보 가능. 실제 시연 현장에서도 UI 개선 니즈 확인

CAUTION

기능 과잉 경계

베가스의 불필요 기능 과다 사례를 반면교사로, 핵심 기능 중심의 정제된 제품 전략 유지 필요
