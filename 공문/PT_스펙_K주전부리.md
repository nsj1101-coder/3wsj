## ▌ 전체 기술 사양

```
파일 형식     : 단일 HTML 파일 (index.html), 외부 CDN만 사용
레이아웃      : 풀스크린 원페이지 (100vw × 100vh), 슬라이드 총 16장
네비게이션    : 방향키(← →), 스페이스바(다음), 마우스 휠, 터치 스와이프
전환 효과     : 슬라이드별 고유 트랜지션 (하단 명세 참조)
현재 슬라이드 표시: 우하단 프로그레스 바 + 슬라이드 번호 (예: 03 / 16)
의존 라이브러리 (CDN):
  - Three.js r128 (https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js)
  - Chart.js 4.x (https://cdn.jsdelivr.net/npm/chart.js)
  - GSAP 3.x (https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js)
  - Pretendard 폰트 (https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/static/pretendard.css)
  - Font Awesome 6 (아이콘용)
```

## ▌ 슬라이드 전환 시스템

```
기본 전환 (다음 슬라이드):
  - 현재 슬라이드: translateX(0) → translateX(-8%) + opacity(1→0) + scale(0.96)
  - 다음 슬라이드: translateX(100%) → translateX(0) + opacity(0→1) + scale(1)
  - duration: 600ms, easing: cubic-bezier(0.76, 0, 0.24, 1)

이전 슬라이드:
  - 방향 반전

특수 전환 (슬라이드별 명시):
  - FLIP: 카드가 3D로 뒤집히며 전환 (rotateY 90°)
  - RISE: 아래에서 위로 슬라이드 인
  - ZOOM: 중앙에서 확대되며 등장
  - FADE: 단순 교차 페이드
```

## ▌ 참고 사항 (다른 AI에게 전달)

```
1. 폰트: Pretendard CDN 사용. 한글 PT이므로 모든 텍스트에 적용 필수.

   Three.js 캔버스는 position:fixed로 배경에 깔고, 나머지 슬라이드에서는 display:none.

3. GSAP: 각 슬라이드 진입 시 해당 슬라이드의 GSAP 타임라인 재생.
   슬라이드 이탈 시 GSAP 타임라인 reverse() 또는 kill().

4. Chart.js: 각 차트는 슬라이드 진입 시 초기화 (canvas ID별 관리).
   이미 초기화된 차트는 destroy() 후 재생성 방지.

5. 전체 HTML은 단일 파일. 외부 이미지 없이 CSS/SVG/Canvas만으로 구현.
   아이콘은 Font Awesome CDN.

6. 반응형: 최소 1280px × 720px 기준. vw/vh 단위 적극 활용.

7. 성능: will-change: transform 적용. 파티클은 requestAnimationFrame 루프.

8. 접근성: 각 슬라이드에 aria-label 추가. 키보드 포커스 관리.

9. 슬라이드 13 (카카오 협력 요청)은 이 PT의 클라이맥스.
   다른 슬라이드보다 시각적으로 한 단계 더 임팩트 있게 제작.

10. 카카오 옐로우 (#FEE500)는 강조 포인트에만 사용. 남용 금지.
    배경이나 대면적에 쓰면 안 됨 — 소량 고강도로 사용할 것.
```
